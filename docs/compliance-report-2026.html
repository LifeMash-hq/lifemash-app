Critical100+ 한국 언론사 RSS 무허가 상업적 수집

NewsCollector는 feeds.csv에 등록된 경향신문·동아일보·조선일보·중앙일보·한겨레·KBS·SBS·JTBC·MBC 등 100여 개 한국 언론사의 RSS 피드를 GitHub Actions(하루 2회, UTC 09:30·21:30)로 자동 수집하여 Firestore articles 컬렉션에 저장합니다.

LifeMash-NewsCollector/.github/workflows/collect_rss_news_to_firestore.yml ← 자동화 스케줄

저장 필드: publisher, title, summary, link, image, publishedAt, categories, keywords — 이 중 title·summary·image는 언론사 저작물에 해당하는 정보입니다.

"뉴스저작물을 서버에 저장하고 게시하는 사업자에게 적용되는 징수율: 뉴스저작물 개수 × 단일 판매가격의 75%"
— 한국언론진흥재단 뉴스저작물 사용료 징수규정 (2026년 기준)

RSS는 공개 배포 형식이나, 이를 상업적 플랫폼에서 대량으로 재가공·큐레이션하는 것은 RSS 이용약관 및 저작권법의 허용 범위를 초과합니다. 대부분의 한국 언론사 RSS 이용약관은 개인·비상업적 이용으로 제한합니다. 현재 상태는 KPF 저작권 침해 경고 및 손해배상 청구 대상이 될 수 있습니다.

주의robots.txt 미확인 — 부정경쟁방지법·업무방해 리스크

수집기는 robots.txt를 파싱하거나 Disallow 규칙을 확인하지 않습니다. 일부 언론사는 robots.txt에서 자동화 봇의 RSS 경로 접근을 차단하고 있습니다. 이를 무시한 수집은 법적으로 두 가지 위험을 초래합니다.

① 잡코리아-사람인 판례(대법원 2017)에서 법원이 집중한 것은 기술적 차단을 우회하는 악의적 행위였습니다. robots.txt 무시는 이와 유사하게 평가될 수 있습니다.
② 무차별 요청이 서버 부하를 유발하면 형법상 업무방해죄 및 정보통신망법 위반 문제가 발생합니다.

collector.py 현재 구현:
✓ If-None-Match (ETag) 헤더 — 구현됨
✓ If-Modified-Since 헤더 — 구현됨
✓ 도메인별 2~3초 지연(throttle_host) — 구현됨
✓ User-Agent: LifeMashNewsBot/1.0 — 구현됨
✗ robots.txt 파싱·준수 — 미구현
ETag를 준수하는 크롤러는 추후 소송에서 "원저작권자 서버 자원을 존중했다"는 선의(Good Faith)를 입증하는 물적 증거로 작용하여 부정경쟁행위의 고의성을 조각하는 데 유리하다.
— 본 검토 참조 문서, 6장 크롤링 기술 에티켓

ETag·지연·식별 User-Agent 구현은 선의 입증에 유리한 강점입니다. robots.txt 준수를 추가하면 기술적 에티켓이 완성됩니다.

양호ETag · Rate Limit · 식별 User-Agent — 기술적 선의 입증 요소 충족

수집기는 조건부 HTTP 요청(ETag·If-Modified-Since), 도메인별 랜덤 2~3초 지연, 3회 재시도 지수 백오프, 피드당 50건 상한을 구현하고 있습니다. 이는 잡코리아-사람인 사건에서 제재받은 VPN 우회·무차별 수집과 정반대의 기술적 구조이며, 법적 분쟁 시 선의를 입증하는 핵심 근거가 됩니다.

즉시주요 수집 대상 언론사 상위 20개사에 RSS 상업적 이용 허가 또는 KPF 라이선스 계약 협의 착수

주의FCM 토큰 Firestore 저장 — 계정 탈퇴 시 파기 연동 필요

FCM 토큰은 userId·keywords와 함께 Firestore fcm_tokens 컬렉션에 저장됩니다. Firebase Functions의 notifyKeywordMatch는 FCM 등록 해제된(unregistered) 토큰을 자동 삭제합니다 — 이 부분은 양호합니다.

그러나 계정 탈퇴 시 fcm_tokens 문서 삭제가 Backend API와 연동되어 있는지 확인이 필요합니다. 탈퇴 후에도 토큰이 잔류하면 개인정보보호법 제21조(파기 의무) 위반이 됩니다.

LifeMash-Functions/functions/src/index.ts (notifyKeywordMatch) ← invalid 토큰 자동 삭제 ✓
LifeMash-Backend/src/.../auth ← 탈퇴 API와 fcm_tokens 삭제 연동 여부 확인 필요

종합 평가

LifeMash-hq 전체 레포지터리 검토 결과, 두 가지 즉각적인 법적 위반 상태가 확인됩니다. 첫째, LifeMash-NewsCollector가 100여 개 한국 언론사의 RSS를 개별 라이선스 계약 없이 상업적 서비스에 수집·저장하고 있습니다. 이는 KPF 징수 규정 및 저작권법 위반의 직접 대상입니다. 둘째, 앱에서 Firebase Analytics·Crashlytics를 사용자 동의 없이 수집 중입니다.

양호한 구조도 확인됩니다. AI 어시스턴트는 기사 내용을 컨텍스트로 사용하지 않아 RAG 저작권 침해 위험이 없습니다. 기사 전문은 외부 WebView로 전환하는 딥링크 구조이며, NewsCollector는 ETag·지연·식별 User-Agent 등 기술적 선의를 갖추고 있습니다. 개인정보처리방침도 이미 발행되어 주요 제3자 처리자가 공개되어 있습니다.

KPF 미디어 스타트업 지원사업(2026년 3~4월 공모 중)은 뉴스 데이터 라이선스 문제를 구조적으로 해결하면서 3,000만 원의 자금 지원과 빅카인즈 API 2년 무상 사용이라는 이점을 동시에 제공합니다. 데이터 출처 합법화를 위한 최우선 경로로 검토를 권장합니다.
