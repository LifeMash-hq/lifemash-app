죄송합니다. 현재 뉴스 서비스에 일시적인 문제가 있는 것 같습니다. 대신 오늘 일정을 확인해보겠습니다:

문제 1 — 홈 화면과 AI가 따로 논다

홈에서 뉴스 블록은 잘 보여주는데, AI에게 물어보면 별도의 NewsTool이 Firestore를 직접 뒤진다. 같은 뉴스인데 두 곳에서 따로 관리.

문제 2 — 새 기능마다 백엔드를 고쳐야 한다

날씨 기능 추가? WeatherTool 코드 작성 → 백엔드 배포. 주식 기능? StockTool 작성 → 또 배포. 서비스가 늘 때마다 백엔드에 코드를 하드코딩해야 한다.

문제 3 — 블록을 설치해도 AI가 모른다

외부 개발자가 날씨 블록을 만들어 마켓에 올려도, 사용자가 AI에게 "오늘 날씨"를 물으면 AI는 그 블록의 존재를 모른다.

두 계층 구조

네이티브 핵심 — 캘린더, 그룹처럼 앱에 내장된 기능. 네이티브 UI + AI Tool이 백엔드에 함께 있다.

블록 개발자가 "날씨 블록"을 만든다

2

마켓플레이스에 등록한다

LifeMash 백엔드 코드 수정 없이, 마켓에 올리기만 하면 된다

3

사용자가 설치한다

홈에 날씨 카드가 뜨고, AI에게 "오늘 날씨" 물으면 자동으로 답변 가능

6. 왜 이게 되는가 — MCP 원리
이미 LifeMash 안에 절반은 구현되어 있다

MCP (Model Context Protocol) 원리

AI (Claude)
어떤 도구가 있지?
→
MCP Client
(LifeMash 백엔드)
설치된 블록 목록 확인
→
MCP Server
(각 블록의 서버)
도구 실행 & 결과 반환

AI가 혼자 못하는 일 → 외부 도구 호출 → 결과 받아서 답변
LifeMash의 ExternalToolExecutor가 이미 이 구조

이미 있는 것

ExternalToolExecutor — 블록이 등록한 도구를 AI가 호출하는 코드가 이미 있다.
toolDefinitions — 블록이 Tool 정의를 가질 수 있는 필드가 이미 있다.
executionUrl — 도구 실행 엔드포인트 개념이 이미 있다.

남은 일: 뉴스를 블록으로 분리(NewsTool 제거)하고, 시스템 프롬프트를 개선하는 것. 캘린더/그룹은 네이티브 내장 Tool로 유지.

한 줄 요약

핵심 기능(캘린더, 그룹)은 네이티브로 유지하고,
외부 서비스(뉴스, 날씨, 주식...)는 블록으로 분리한다.

블록은 화면(WebView)과 AI 도구(Tool)를 함께 등록하므로,
설치하는 순간 홈에도 뜨고, AI도 그 기능을 쓸 수 있게 된다.
