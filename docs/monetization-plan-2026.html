LifeMash 수익화 방안

기준일: 2026년 3월 20일 전제: 사업자 등록 불가 / 수익은 필요

💡 먼저 알아야 할 것 — "사업자 등록 불가" ≠ "수익화 불가"

많은 분들이 혼동하지만, 앱 수익화에 사업자 등록은 법적 필수 요건이 아닙니다. 구글 플레이·앱스토어 개발자 계정은 개인 자격으로 개설 가능하고, AdMob 광고 수익은 개인 계좌로 받을 수 있습니다. 수익은 기타소득 또는 사업소득으로 연말에 종합소득세 신고하면 됩니다.

문제는 겸직 금지 조항입니다. 대부분 회사의 겸직 금지는 "타 회사 취업" 또는 "본업에 지장을 주는 활동"을 금지하는 것이지, 개인 앱 출시 자체를 금지하는 경우는 많지 않습니다. 회사 HR 또는 취업규칙을 직접 확인하는 것이 먼저입니다. 소규모 개인 앱 운영을 문제 삼는 경우는 현실적으로 드뭅니다.

단, 광고 수익이 생기는 순간 "비상업적 서비스" 방패는 사라지므로, 뉴스 콘텐츠 저작권 문제를 동시에 해결해야 합니다. 이 두 가지를 묶어서 해결하는 방안을 아래에 제시합니다.

2수익화와 동시에 해결 — 뉴스 소스 교체 (저작권 클리어)

AdMob이나 인앱결제를 도입하면 상업적 서비스가 됩니다. 이 시점에 현재 RSS 수집 구조는 저작권 리스크를 수반합니다. 아래 두 가지 방법으로 비용 없이 해결할 수 있습니다.

🏛️
최우선 추천 무료·상업 이용 가능 사업자 불필요
공공저작물 뉴스 API — 무료이면서 상업적 이용 완전 허용

대한민국 공공저작물은 출처 표기만 하면 상업적 이용이 자유롭습니다 (공공누리 1유형). 개인 신청 가능하며 별도 계약이 필요 없습니다.

연합뉴스 공공 API
data.go.kr 등록. 연합뉴스 공식 기사. 하루 수천 건.
뉴스 빅데이터 (공공)
한국언론진흥재단 공공 API. 정부 신청으로 무료.

⚠️ 단기 현실: 공공 API의 기사 종류와 양은 현재 RSS 100+ 언론사보다 적습니다. 그러나 법적으로 완전히 클리어된 상태에서 수익화가 가능하다는 것이 핵심입니다. 사용자 수가 적은 초기에 소스를 교체해두면 나중에 훨씬 편합니다.

1data.go.kr에서 "뉴스" 검색 → 연합뉴스, 뉴시스 등 공공 뉴스 API 신청 (개인 신청 가능, 1~2일 승인)
2NewsCollector의 RSS 수집 로직을 공공 API 호출로 교체 (이미 추상화 준비 중)
3기사 표출 시 "출처: 연합뉴스" 등 공공누리 출처 표기 추가
⚡
즉시 적용 가능 기존 구조 유지
Firestore summary·image 제거 + 원본 링크만 표시 — 복제 최소화

공공 API 전환 전 임시 방편으로, Firestore에서 summary와 image를 삭제하고 피드에는 제목+언론사+날짜+링크만 표시합니다. 클릭 시 원본 기사로 이동. 저장되는 저작물이 최소화되고, 원본 서버로 트래픽을 100% 유도하는 구조가 됩니다.

이 구조에서는 서버에 복제된 저작물이 사실상 없으므로 법적 공세 자체가 성립하기 어렵습니다. 완벽한 해결책은 아니지만 공공 API 전환 전까지의 현실적인 완충재입니다.

3겸직 문제 실용 조언 — 확인하고 넘어가야 할 것
📋
먼저 확인 필수
취업규칙·근로계약서에서 확인해야 할 것

겸직 금지 조항의 실제 내용이 중요합니다. 대부분 아래 중 하나입니다:

A"타 회사에 취업하거나 임원이 될 수 없다" → 개인 앱 수익은 해당 없음. 수익화 가능.
B"회사의 사전 허락 없이 영리 활동을 할 수 없다" → HR에 앱 개인 프로젝트 운영 의향 비공식 문의. 대부분 허용.
C"어떠한 형태의 부업도 금지" → 배우자 명의 사업자 등록 검토. 또는 수익화를 사업자 등록 가능 시점까지 보류하고 AI 어시스턴트 무료 → 사용자 확보 우선.

현실적으로: 소규모 개인 앱으로 월 수십만원 수준의 부수입을 얻는 것을 문제 삼아 징계하는 회사는 극히 드뭅니다. 법적으로도 근로자의 퇴근 후 활동은 원칙적으로 자유입니다. 단, 회사 기밀이나 본업 경쟁 서비스에 해당한다면 별개입니다.

결론

지금 당장 할 수 있는 것: 설정 화면에 후원 링크 하나 추가하는 것부터 시작하세요. 이건 오늘 30분이면 됩니다.

한 달 안에 의미 있는 수익을 만들려면: Firestore summary·image 제거(저작권 리스크 제거) → AdMob 도입 → 공공 뉴스 API 전환, 이 순서가 가장 현실적입니다.

중장기 수익 핵심: AI 어시스턴트 프리미엄이 가장 명분 있는 수익화입니다. "Claude API 비용이 드니까 유료"라는 논리가 사용자에게도, 법적으로도 가장 깔끔합니다. 뉴스는 사용자 유입 채널로 두고, 수익은 AI 기능으로 얻는 구조가 LifeMash의 방향성과도 맞습니다.
